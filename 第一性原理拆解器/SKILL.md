# 第一性原理拆解器

## 何时使用
- 设计新产品或功能
- 在复杂问题上感到困惑
- 现有解决方案看起来过于复杂
- 需要质疑假设
- 开始任何新项目或计划

## 流程

### 阶段1：识别假设
问："我假设什么是真的，但可能不是？"
列出当前方法中嵌入的每一个假设。

### 阶段2：拆解到原子
对每个假设，问："这里最基本的事实是什么？"
不断问"为什么？"直到触及基础事实。

### 阶段3：从真理重建
仅从已验证的基本事实出发，问：
"解决核心需求的最简单方案是什么？"

## 交互流程

当用户调用此技能时：

1. **澄清问题**（最多1-2个问题）
2. **浮现假设** - 列出被视为理所当然的内容
3. **分解到基本原理** - 展示原子级真理
4. **重建解决方案** - 从零开始构建
5. **对比** - 展示这与传统方法的不同

## 输出格式

```
问题：[陈述的问题]

识别的假设：
1. [假设] → 质疑：[为什么这可能是错的]
2. [假设] → 质疑：[为什么这可能是错的]

基本真理：
• [基础事实1]
• [基础事实2]
• [基础事实3]

重建的解决方案：
[仅基于基本原理的新方法]

与传统方法对比：
[这与显而易见的方法有何不同]
```

## 示例触发词
- "从第一性原理拆解我们的家长沟通问题"
- "我想从零开始重新思考我们如何做[X]"
- "关于[问题]，我们在假设什么可能是错的？"

## 集成

此技能可与以下技能组合：
- **逆向策略师** - 从基本原理重建后，逆向思考什么会导致新方案失败
- **二阶后果分析** - 预测实施重建方案后的下游影响
- **预演分析师** - 通过想象失败来压力测试重建方案
- **六顶思考帽** - 应用六个视角验证每个识别的基本真理
