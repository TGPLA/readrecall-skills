# 本地仓库分析器运行准则

## 1. 触发前置 (上下文获取)
在开始任何修改前，你必须向用户确认参考项目的**本地绝对路径**。
- **指令**：请用户通过 `@Folder` 或直接提供路径指向参考项目。
- **动作**：优先阅读参考项目的 `README.md`、`package.json`（或环境配置文件）以及核心逻辑文件夹。

## 2. 感知阶段 (感知) - 深度拆解
针对用户提出的修改需求，在参考项目中寻找对应的实现：
- **搜索逻辑**：在参考项目中使用 `grep` 或内置搜索，寻找相似的功能模块、API 定义或 UI 组件。
- **逻辑提取**：分析参考项目是如何处理该功能的边缘情况（边界情况）和数据流的。

## 3. 规划阶段 (规划) - 映射与适配
禁止直接复制代码。你需要输出一份"重构报告"：
- **参考点**：参考项目中哪些文件/函数可以直接借鉴？
- **差异点**：参考项目的架构与当前项目有何冲突？需要做哪些改动才能"丝滑搬运"？
- **依赖检查**：参考项目使用了哪些三方库？当前项目是否需要同步安装？

## 4. 执行阶段 (执行) - 智能复刻
- **代码移植**：按照参考项目的逻辑模式，在当前项目中重写或粘贴代码。
- **规范统一**：将参考项目的变量命名、缩进风格、类型定义自动转化为当前项目的规范。

## 5. 反思阶段 (反思) - 优化验证
- 检查移植后的功能是否比原参考项目更简洁。
- 确认是否遗漏了参考项目中的关键注释或错误处理逻辑。
