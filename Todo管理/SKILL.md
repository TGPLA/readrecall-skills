# Todo 管理

## 功能说明

基于 JSON 文件的待办事项管理器，存储在 `./data/todos.json`

---

## 数据结构

```json
{
  "entries": [
    {
      "id": 1,
      "text": "待办内容",
      "status": "pending|in_progress|done|skipped",
      "group": "分组名",
      "createdAt": "ISO时间",
      "updatedAt": "ISO时间"
    }
  ],
  "groups": ["分组1", "分组2"],
  "nextId": 2
}
```

---

## 状态说明

| 状态 | 含义 |
|------|------|
| `pending` | 待处理（默认） |
| `in_progress` | 进行中 |
| `done` | 已完成 |
| `skipped` | 已跳过 |

---

## 核心规则

### 1. 显示规则
- 用户**未要求**显示列表时，**不要主动显示**
- 添加/完成/删除后只回复简短确认（一行）
- 允许的确认语："✅ 已添加"、"✅ 已完成"、"🗑️ 已删除"、"📝 已更新"

### 2. 歧义处理
- 用户指定删除但未提供 ID → 先列出匹配项 → 询问具体 ID

### 3. 数据文件
- 位置：`./data/todos.json`
- 不存在时自动创建

## 命令说明

### 添加待办
```
添加待办"内容"
添加待办"内容"到"分组"
添加待办"内容"并标记为进行中
```

### 查看待办
```
查看待办列表
显示所有待办
查看"分组"分组下的待办
查看进行中的待办
```

### 完成待办
```
完成第 N 项
把第 N 项标记为完成
```

### 删除待办
```
删除第 N 项
删除"分组"分组（确认是否连同待办一起删除）
```

### 分组管理
```
创建分组"分组名"
重命名分组"旧名"为"新名"
删除分组"分组名"（可选择是否删除待办）
```

## 工作流程

### 添加待办
1. 读取 `./data/todos.json`（不存在则创建）
2. 生成新 ID
3. 添加条目
4. 写入文件
5. 回复："✅ 已添加"

### 查看列表
1. 读取文件
2. 筛选状态/分组
3. 格式化显示
4. 回复列表

### 完成待办
1. 读取文件
2. 找到对应 ID
3. 更新状态为 `done`
4. 写入文件
5. 回复："✅ 已完成"

### 删除待办
1. 读取文件
2. 过滤掉对应 ID
3. 重新编号
4. 写入文件
5. 回复："🗑️ 已删除"

## 显示格式

```
📋 待办列表（共 N 项）

[1] ⏳ 待办内容 A - 工作
[2] 🔄 待办内容 B - 生活
[3] ✅ 已完成内容 C - 工作

显示说明：⏳ pending  🔄 in_progress  ✅ done  ⏭️ skipped
```
