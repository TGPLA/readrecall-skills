# 失败复盘存档

当同一问题连续尝试修复 3 次以上仍未解决时，自动触发此流程。

## 触发条件

- 同一问题连续修复 3 次以上
- 每次修复后用户反馈"还是不行"或类似表述
- 问题没有明显进展或反复出现

## 执行步骤

### 步骤 1：核心矛盾总结

用中文详细说明为什么过去几轮尝试都失败了？

分析维度：
- **逻辑冲突**：代码逻辑是否存在矛盾？
- **环境问题**：是否是浏览器/系统/版本差异导致？
- **库兼容性**：是否是某个库的版本不兼容？
- **理解偏差**：是否对问题的理解有误？

### 步骤 2：排除法记录

列出已经证明**"行不通"**的代码方案，格式如下：

```markdown
## 排除法记录

| 尝试次数 | 方案描述 | 失败原因 |
|----------|----------|----------|
| 第1次 | 方案A | 具体原因 |
| 第2次 | 方案B | 具体原因 |
| 第3次 | 方案C | 具体原因 |
```

### 步骤 3：保留火种

是否有哪一版的部分代码是正确的？请整理出来：

```markdown
## 保留火种

### 可复用的代码片段
- 片段1：描述 + 代码
- 片段2：描述 + 代码

### 正确的思路方向
- 思路1
- 思路2
```

### 步骤 4：同步更新文档

更新以下文件：
1. `08_项目进度.md` - 在『未竟之事』中添加排除记录
2. `02_项目地图.md` - 如有需要更新相关说明

### 步骤 5：生成求助式接力语

为新窗口写一段开场白，格式如下：

```markdown
## 求助式接力语

**问题描述**：[简洁描述问题]

**已尝试但失败的方案**：
1. 方案A - 失败原因
2. 方案B - 失败原因
3. 方案C - 失败原因

**请避开以上方案，重新审视问题**：
- 建议从 [新角度] 入手
- 注意 [关键点] 可能是突破口

**相关文件**：
- [文件路径1]
- [文件路径2]
```

## 输出要求

完成所有步骤后，将『求助式接力语』发送给用户，方便用户在新窗口继续解决问题。
